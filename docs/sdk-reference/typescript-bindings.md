# TypeScript SDK Reference: Frontend Bindings

This document describes the complete TypeScript API for the GP2F frontend SDK. The SDK is distributed as an ES module in the `client-sdk` package and provides typed wrappers around the WASM evaluator, the WebCrypto KMS integration, the `op_id` generation protocol, the offline sync manager, and the fluent policy builder.

## Module: `policy-builder`

The `policy-builder` module exports the **Fluent Policy Builder** API for constructing policy ASTs without writing raw JSON. Import with:

```typescript
import { p, PolicyBuilder, FieldBuilder, VibeBuilder } from '@gp2f/client-sdk';
```

`p` is a shorthand alias for `PolicyBuilder`. All builder methods are static.

### Quick example

```typescript
import { p } from '@gp2f/client-sdk';

const policy = p.and(
  p.or(
    p.field('/user/role').eq('admin'),
    p.field('/user/role').eq('editor'),
  ),
  p.exists('/session/token'),
  p.not(p.field('/account/suspended').eq('true')),
);
```

See the full [Fluent Policy Builder Reference](policy-builder.md) for all available operators (`field`, `and`, `or`, `not`, `exists`, `literalTrue`, `literalFalse`, `vibe`, comparison operators, and collection operators).

## Module: `crypto/kms`

The KMS module manages cryptographic keys using the WebCrypto API. Keys are generated on device, stored in a non-exportable `CryptoKey` object in memory, and persisted across sessions using the `IndexedDB` key store.

### `initKms(sessionId: string): Promise<void>`

Initializes the KMS for the given session. If a key for this session exists in the IndexedDB key store, it is loaded. If not, a new key pair is generated.

```typescript
import { initKms } from 'client-sdk/crypto/kms';

await initKms('sess_xyz789');
```

This function must be called before any function that uses cryptographic keys (`generateOpId`, `enqueueOp`, `dequeueOp`). It is idempotent for a given `sessionId`.

### `getSessionKey(): Promise<CryptoKey>`

Returns the HMAC-SHA256 `CryptoKey` for the current session. Used by `generateOpId` to sign operation payloads.

```typescript
import { getSessionKey } from 'client-sdk/crypto/kms';

const key: CryptoKey = await getSessionKey();
// key.type === 'secret', key.algorithm.name === 'HMAC'
```

Throws `KmsError('NOT_INITIALIZED')` if `initKms` has not been called.

### `getEncryptionKey(): Promise<CryptoKey>`

Returns the AES-GCM `CryptoKey` used to encrypt and decrypt IndexedDB queue records.

```typescript
import { getEncryptionKey } from 'client-sdk/crypto/kms';

const encKey: CryptoKey = await getEncryptionKey();
// encKey.type === 'secret', encKey.algorithm.name === 'AES-GCM'
```

The HMAC key and the AES-GCM key are derived from the same root key material using HKDF with distinct info strings, ensuring cryptographic domain separation.

### `exportPublicSessionId(): string`

Returns the public session identifier: a base64url-encoded hash of the HMAC key material. This is safe to transmit to the server and is used to look up the session in the Postgres `sessions` table.

## Module: `crypto/opId`

### Interface: `OpIdPayload`

```typescript
export interface OpIdPayload {
  opId: string;           // Hex-encoded HMAC-SHA256 signature
  sessionId: string;      // Public session identifier
  intent: Intent;         // The user's declared action intent
  clientStateHash: string;// BLAKE3 hex hash of the PolicyState at time of generation
  timestampMs: number;    // Unix timestamp in milliseconds
  sequenceNumber: number; // Monotonically increasing per-session counter
}
```

### `generateOpId(intent: Intent, clientStateHash: string): Promise<OpIdPayload>`

Generates a cryptographically signed `op_id` for the given intent and state hash.

```typescript
import { generateOpId } from 'client-sdk/crypto/opId';

const payload = await generateOpId(
  { actionId: 'submit_document', documentId: 'doc_456' },
  'b7e2aa3c9f...' // BLAKE3 hash of current PolicyState
);

console.log(payload.opId);
// "a3f9bc7e2d8c1a4b5e6f9012345678901234567890abcdef0123456789abcdef01"
```

The `opId` field is computed as:

```
opId = hex(HMAC-SHA256(key=sessionKey, data=CBOR(intent || timestampMs || clientStateHash || sequenceNumber)))
```

The CBOR encoding is used for the data payload to ensure canonical serialization across different environments.

**Throws:** `KmsError('NOT_INITIALIZED')` if the KMS has not been initialized. `CryptoError('SIGN_FAILED')` if the WebCrypto sign operation fails.

## Module: `crypto/hash`

### `blake3Hash(data: string | Uint8Array): Promise<string>`

Computes the BLAKE3 hash of the input and returns a 64-character hex string. Used to compute the `clientStateHash` for `op_id` generation.

```typescript
import { blake3Hash } from 'client-sdk/crypto/hash';

const hash = await blake3Hash(JSON.stringify(policyState));
// "a3f9bc7e2d8c1a4b5e6f901234567890..."
```

The BLAKE3 implementation is the `blake3` WASM package, which is loaded lazily alongside the policy evaluator. The hash function is deterministic: given the same serialized state, it will always produce the same hash.

**Note on serialization:** Canonical JSON serialization (keys sorted alphabetically, no extra whitespace) is required for the `clientStateHash` to be reproducible across the client and server. Use `JSON.stringify` with a replacer or a canonical serializer library.

## Module: `queue/indexedDbQueue`

### `enqueueOp(payload: OpIdPayload): Promise<void>`

Encrypts the `OpIdPayload` with AES-GCM and writes it to the IndexedDB `pending_ops` store.

```typescript
import { enqueueOp } from 'client-sdk/queue/indexedDbQueue';

await enqueueOp(opIdPayload);
// The record is now durably stored and will survive page refresh
```

The encrypted record has the following structure in IndexedDB:

```typescript
interface QueueRecord {
  opId: string;        // Plaintext key (not sensitive)
  iv: number[];        // AES-GCM IV as a number array
  ciphertext: number[];// Encrypted OpIdPayload as a number array
  enqueuedAt: number;  // Unix timestamp in milliseconds
}
```

The `iv` is generated fresh for each record using `crypto.getRandomValues`. It is safe to store in plaintext.

### `dequeueOp(opId: string): Promise<void>`

Removes the record with the given `opId` from the IndexedDB queue. Called after the server confirms an `OP_ACK` for the operation.

```typescript
import { dequeueOp } from 'client-sdk/queue/indexedDbQueue';

await dequeueOp('a3f9bc7e2d8c...');
```

### `getAllPendingOps(): Promise<OpIdPayload[]>`

Returns all records from the IndexedDB queue, decrypted and deserialized. Used on WebSocket reconnection to replay all pending operations.

```typescript
import { getAllPendingOps } from 'client-sdk/queue/indexedDbQueue';

const pending = await getAllPendingOps();
console.log(`Replaying ${pending.length} pending operations`);
for (const op of pending) {
  socket.send(JSON.stringify({ type: 'OP_ID', payload: op }));
}
```

### `getQueueDepth(): Promise<number>`

Returns the number of records currently in the queue. Useful for rendering an offline sync indicator.

```typescript
import { getQueueDepth } from 'client-sdk/queue/indexedDbQueue';

const depth = await getQueueDepth();
if (depth > 0) {
  showOfflineSyncBadge(depth);
}
```

## Module: `sync/syncManager`

The `SyncManager` manages the WebSocket connection lifecycle, including exponential backoff reconnection, message routing, and offline queue replay.

### `createSyncManager(options: SyncManagerOptions): SyncManager`

```typescript
import { createSyncManager } from 'client-sdk/sync/syncManager';

const syncManager = createSyncManager({
  url: 'wss://api.example.com/ws',
  sessionId: 'sess_xyz789',
  onOpAck: (opId, outcome, canonicalPatch) => {
    store.confirmOp(opId, outcome, canonicalPatch);
    dequeueOp(opId);
  },
  onAstUpdate: (ast, version) => {
    store.loadAst(ast, version);
  },
  onStateSync: (state) => {
    store.loadState(state);
  },
  onConnectionChange: (connected) => {
    setIsOnline(connected);
  },
});

await syncManager.connect();
```

### Interface: `SyncManagerOptions`

```typescript
export interface SyncManagerOptions {
  url: string;
  sessionId: string;
  onOpAck: (opId: string, outcome: 'ACCEPT' | 'REJECT', canonicalPatch: PolicyState | null) => void;
  onAstUpdate: (ast: AstNode, version: string) => void;
  onStateSync: (state: PolicyState) => void;
  onConnectionChange: (connected: boolean) => void;
  reconnectDelayMs?: number;    // Default: 1000
  maxReconnectDelayMs?: number; // Default: 30000
  reconnectJitterMs?: number;   // Default: 500
}
```

### `SyncManager.connect(): Promise<void>`

Establishes the WebSocket connection. Returns when the connection is open. Automatically replays all pending operations from the IndexedDB queue on connection.

### `SyncManager.emit(message: ClientMessage): void`

Sends a message over the WebSocket. If the connection is not open, the message is buffered and sent on reconnection.

```typescript
syncManager.emit({
  type: 'OP_ID',
  payload: opIdPayload,
});
```

### `SyncManager.disconnect(): void`

Closes the WebSocket connection and stops reconnection attempts. Does not clear the IndexedDB queue.

### `SyncManager.getConnectionState(): 'connected' | 'connecting' | 'disconnected'`

Returns the current connection state.

## Module: `stores/policyStore`

The Zustand policy store is described in detail in the React Client Integration guide. The following summarizes the exported API.

### `usePolicyStore`

```typescript
import { usePolicyStore } from 'client-sdk/stores/policyStore';
```

Returns a Zustand store with the following interface:

```typescript
interface PolicyStoreState {
  currentAst: AstNode | null;
  astVersion: string | null;
  canonicalState: PolicyState | null;
  optimisticState: PolicyState | null;
  permittedActions: string[];
  pendingOps: Map<string, OptimisticEntry>;

  loadAst: (ast: AstNode, version: string) => void;
  loadState: (state: PolicyState) => void;
  evaluateIntent: (intent: Intent) => EvalResult | null;
  applyOptimisticUpdate: (opId: string, intent: Intent, predictedState: PolicyState) => void;
  confirmOp: (opId: string, outcome: 'ACCEPT' | 'REJECT', canonicalPatch: PolicyState | null) => void;
}
```

**`loadAst`** — Loads a new AST and re-evaluates permitted actions for the current state. Triggers a re-render in all components that consume `permittedActions`.

**`loadState`** — Sets the canonical and optimistic state. Re-evaluates permitted actions.

**`evaluateIntent`** — Calls the WASM evaluator synchronously and returns an `EvalResult`. Returns `null` if the WASM evaluator is not initialized or if the AST or state is not loaded.

**`applyOptimisticUpdate`** — Records a pending operation and updates the `optimisticState`. Call this immediately after generating an `op_id`, before network emission.

**`confirmOp`** — Processes a server acknowledgment. On `ACCEPT`, finalizes the optimistic update. On `REJECT`, rolls back to the pre-operation state and replays any subsequent pending operations.

## Core Type Definitions

```typescript
// src/types/policy.ts

export interface Intent {
  actionId: string;
  documentId: string;
  [key: string]: unknown;
}

export interface EvalResult {
  permitted: boolean;
  trace: string[];
  snapshotHash: string;
}

export interface PermittedActionsResult {
  permittedActions: string[];
  snapshotHash: string;
}

export interface PolicyState {
  documentId?: string;
  fields: Record<string, unknown>;
  vectorClock: Record<string, number>;
}

export interface AstNode {
  nodeType: string;
  children?: AstNode[];
  field?: string;
  value?: string | number | boolean | null | AstNode[];
  actionId?: string;
}

export interface OptimisticEntry {
  opId: string;
  previousState: PolicyState;
  predictedState: PolicyState;
  intent: Intent;
  emittedAt: number;
}

export type ClientMessage =
  | { type: 'OP_ID'; payload: OpIdPayload }
  | { type: 'PING' };

export type ServerMessage =
  | { type: 'OP_ACK'; payload: { opId: string; outcome: 'ACCEPT' | 'REJECT'; canonicalPatch: PolicyState | null } }
  | { type: 'AST_UPDATE'; payload: { ast: AstNode; version: string } }
  | { type: 'STATE_SYNC'; payload: { state: PolicyState } }
  | { type: 'PONG' };
```

## Error Types

```typescript
export class KmsError extends Error {
  constructor(
    public code: 'NOT_INITIALIZED' | 'KEY_GENERATION_FAILED' | 'KEY_LOAD_FAILED',
    message: string
  ) {
    super(message);
    this.name = 'KmsError';
  }
}

export class CryptoError extends Error {
  constructor(
    public code: 'SIGN_FAILED' | 'ENCRYPT_FAILED' | 'DECRYPT_FAILED' | 'HASH_FAILED',
    message: string
  ) {
    super(message);
    this.name = 'CryptoError';
  }
}

export class SyncError extends Error {
  constructor(
    public code: 'CONNECTION_FAILED' | 'AUTH_FAILED' | 'QUEUE_FULL',
    message: string
  ) {
    super(message);
    this.name = 'SyncError';
  }
}
```
