# Project Proposal 6: Autonomous Contract Negotiator (ACN)

## 1. Project Definition
**ACN** is an enterprise legal tech platform where AI agents autonomously negotiate commercial lease agreements on behalf of landlords and tenants. The platform uses **gp2f** as the authoritative ledger and policy enforcer, while **OpenAI's GPT-4o** acts as the "Agent" proposing changes.

Crucially, the AI is **not trusted**. Every proposal generated by the LLM is treated as an untrusted `op` and validated against a rigorous, deterministic `gp2f` Policy (the "Guardrails"). If the AI hallucinates a clause that violates the client's risk tolerance (e.g., "Rent cap: $0"), the policy engine rejects it instantly, preventing the contract from entering an invalid state.

## 2. Objectives
*   **Safe AI Autonomy:** Enable agents to draft and redline contracts without human oversight, guaranteed by mathematical policy constraints.
*   **Speed:** Reduce negotiation time from weeks to minutes.
*   **Audit Trail:** Every AI thought process and proposed diff is cryptographically signed and stored in the `gp2f` event log.
*   **Deterministic Outcome:** Given the same initial contract and policies, the final agreement is reproducible.

## 3. Product Requirements Document (PRD)

### 3.1 Frontend Screens (Legal Dashboard)

#### Screen 1: The Negotiation Room
*   **View:** Split-screen diff of the contract (Current vs. Proposed).
*   **Actors:** "Landlord Agent" (AI) vs. "Tenant Agent" (AI).
*   **Live Feed:** Streaming text of the AI's rationale ("I am rejecting section 4.2 because the liability cap is too low.").
*   **Interaction:** Human override. A lawyer can pause the AI and manually inject an `op`.

#### Screen 2: Policy Configuration (The "Guardrails")
*   **View:** Form-based builder for risk parameters.
*   **Inputs:** "Max Rent Increase: 5%", "Min Lease Term: 3 Years", "Governing Law: NY or DE only".
*   **Output:** Generates a `gp2f` Policy AST that binds the AI's hands.

#### Screen 3: Final Execution
*   **Action:** "Sign Contract".
*   **Validation:** The system runs a final deep-scan using `gp2f` to ensure *all* constraints are met before applying the digital signature.

### 3.2 Backend Architecture (Rust/Axum + OpenAI)

#### Data Model (Proto)
*   **Aggregate:** `Contract`
*   **State:**
    ```protobuf
    message ContractState {
      string id = 1;
      string content_markdown = 2; // The legal text
      map<string, string> clauses = 3; // Key-value extraction
      enum Status { DRAFT = 0; NEGOTIATING = 1; SIGNED = 2; }
    }
    ```

#### Policy Definitions (AST) - The "Risk Manager"
*   **File:** `policies/commercial_lease_v3.json`
*   **Logic (Parsing & Assertions):**
    *   *Clause Extraction:* A regex-based or structural parser extracts "Rent", "Term", "Liability".
    *   *Assertion:* `Assert(Rent.Increase <= 0.05)`.
    *   *Jurisdiction:* `Assert(State IN ["NY", "DE"])`.
*   **Rejection:** If the AI proposes "Rent Increase: 10%", the Policy Engine returns `REJECT: Policy Violation (Max 5%)`. The Agent receives this error and must re-prompt itself to try again.

### 3.3 Implementation Steps

#### Step 1: The Agent Loop (Server-Side)
*   Implement a background worker in Rust that:
    1.  Listens for `ContractUpdated` events.
    2.  Constructs a prompt for OpenAI: "You are the Tenant's lawyer. The current rent is $50/sqft. Your goal is $45. Propose a redline."
    3.  Receives the JSON diff from OpenAI.
    4.  Submits it as a `gp2f` `op` (`ProposeChange`).

#### Step 2: Policy Enforcement
*   The `gp2f` Reconciler intercepts the `ProposeChange` op.
*   It runs the WASM policy against the *proposed* state.
*   If valid -> `ACCEPT`. The contract updates.
*   If invalid -> `REJECT`. The error is fed back to the Agent's context window ("Your proposal was rejected: Rent too high").

#### Step 3: Human-in-the-Loop Override
*   Allow the user (via `client-sdk`) to issue a `PauseNegotiation` op.
*   Allow manual `EditClause` ops that override the AI's latest proposal (LWW strategy).

#### Step 4: Token Usage & Cost Tracking
*   Log every OpenAI API call in a separate `UsageLog` aggregate to track costs per negotiation.
